# Session Notes — 21 April 2026 (cont.)

## Mobile fix: Global Presence & Expert Opinions cards

### Problem
On mobile (≤768px) both cards displayed at 50% of viewport width — too narrow to read map or text.

### Root cause
- `.presence-card` and `.expert-opinions-card` both had `width: 50%` as base rules
- The mobile override added inside the `@media (max-width: 768px)` block at line ~1892 was placed *before* the `.expert-opinions-card` base rule at line 2192 in the file — so the base rule won the cascade
- A stray extra `}` at end of file was also found and removed (CSS bug)

### Fix
- Added a dedicated `@media (max-width: 768px)` block directly after `.expert-opinions-card`'s base rule → `width: 100%`
- `#abilar-map` height reduced from 340px → 240px on mobile
- `index.html` cache buster bumped `v=16` → `v=22`
- Removed stray closing `}` at end of `styles.css`

### Files changed
| File | Change |
|------|--------|
| `styles.css` | Mobile override for `.expert-opinions-card`, map height, stray brace removed |
| `index.html` | CSS version bump to v=22 |
